OTHER FEATURES

Multiple versions

Supporting existing data models while simultaneously allowing new versions, since when new endpoints are developed, the data modes often needs to undergo revisions and refinements.

Ability to share

Ability to share object notations and code. While standard data models can be found from several SDOs, it often becomes advantageous to provide platform-specific code implementations of the managed object.

Other supports

Support for proprietary or non-standard models, the repository can serve as a trusted reference point for management servers to retrieve various models as well as understand the relationships among multiple data models.

Contribute your own

You can create new models and add it to repository for others, contributors are encouraged to design their domain-specific data models using online data models supplied by the relevant SDOs (Standards Developing Organizations).
